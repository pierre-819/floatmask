## 四、遇到的问题与解决过程

### 问题 1：中文字符显示乱码

**现象**：App 界面所有中文按钮和标签显示为方块或乱码。

**根本原因**：Kivy 默认字体 Roboto 不包含中文字符集。

**解决方案**：
1. 从 Windows 系统字体目录（`C:\Windows\Fonts\simsun.ttc`）复制宋体字体文件到项目
2. 在 `main.py` 中用 `LabelBase.register()` 注册字体：
   ```python
   LabelBase.register(name="SimSun", fn_regular="simSun.ttc")
   ```
3. 在 `buildozer.spec` 中添加 `ttc` 到打包扩展名：
   ```
   source.include_exts = py,png,jpg,kv,json,ttc,ttf
   ```

---

### 问题 2：pyjnius 调用 setText() 崩溃

**现象**：调用 `TextView.setText("文字")` 时抛出异常：
```
No methods called 'setText' matching your arguments
```

**根本原因**：pyjnius 在 Python str 与 Java 方法签名匹配时无法自动推断重载版本，需要显式传入 Java 类型。

**解决方案**：将所有字符串参数用 Java String 对象包装：
```python
JString = autoclass("java.lang.String")
hint.setText(cast("java.lang.CharSequence", JString("FloatMask")))
```
同理，`dialog.setTitle()`、`setItems()` 等涉及字符串的 API 均需相同处理。

---

### 问题 3：标题被状态栏遮挡

**现象**：App 顶部标题"FloatMask 智能字幕遮挡工具"被手机状态栏（时间、信号等）覆盖，看不清。

**根本原因**：Kivy BoxLayout 默认 padding 从屏幕左上角 (0,0) 开始，未给状态栏留出空间。

**解决方案**：修改 KV 布局的 padding，顶部增加 48dp：
```
padding: dp(18), dp(48), dp(18), dp(18)
```

---

### 问题 4：授权后点击"显示悬浮框"报 View not attached 错误

**现象**：首次运行授权悬浮窗权限后返回 App，点击"显示/编辑悬浮框"提示：
```
addView失败: View not attached to window manager
```

**根本原因**：`show()` 方法内部缓存了 `_view` 引用。若上一次 `addView` 抛出异常（如权限未就绪），`_view` 已被创建但未成功挂载。下次调用时代码误以为 View 已挂载，跳过重建逻辑，直接调用 `updateViewLayout` 导致该错误。

**解决方案**：在 `addView` 的 `except` 块中清空 `_view` 引用，下次调用时强制重建：
```python
except Exception as e:
    self._view = None
    self._params = None
    raise RuntimeError(f"addView失败: {e}") from e
```
同时在 `show()` 入口增加权限校验，未授权时直接抛出友好提示，不进入 View 创建流程。

---

### 问题 5：Can't create handler inside thread that has not called Looper.prepare()

**现象**：点击"显示/编辑悬浮框"报错：
```
addView失败: JVM exception occurred: Can't create handler inside thread 
Thread[SDLThread,5,main] that has not called Looper.prepare()
java.lang.RuntimeException
```

**根本原因**：Kivy 运行在 SDL 渲染线程（SDLThread）中，而 Android 的 `WindowManager.addView()` / `removeView()` / `updateViewLayout()` 等 UI 操作**必须在 Android 主线程（UI Thread / Main Thread）上调用**，在非主线程调用会因缺少 Looper 而崩溃。

**解决方案**：新增 `_run_on_ui_thread()` 工具函数，通过 `activity.runOnUiThread(Runnable)` 将所有 WindowManager 操作切换到 Android 主线程执行：
```python
def _run_on_ui_thread(activity, fn):
    from jnius import PythonJavaClass, java_method
    import threading
    done = threading.Event()
    exc_box = []

    class Run(PythonJavaClass):
        __javainterfaces__ = ["java/lang/Runnable"]
        __javacontext__ = "app"

        @java_method("()V")
        def run(self):
            try:
                fn()
            except Exception as e:
                exc_box.append(e)
            finally:
                done.set()

    activity.runOnUiThread(Run())
    done.wait()
    if exc_box:
        raise exc_box[0]
```

### 问题 6：vivo 手机额外的厂商权限

**现象**：在系统设置中已开启"悬浮窗"权限，但悬浮框仍无法显示（权限检查 `canDrawOverlays()` 返回 True，但 `addView` 被系统拦截）。

**根本原因**：vivo 手机在 Android 标准权限之上，还有**厂商自定义的"后台弹出界面"权限**，需在 vivo i管家（安全中心）中单独授权。仅开启系统设置中的悬浮窗权限是不够的。

**解决方案**（用户操作步骤）：
1. 打开 **i管家**（vivo 手机预装安全应用）
2. 进入 **应用管理** → 找到 FloatMask
3. 开启 **"后台弹出界面"** 权限

**报告说明**：这属于"厂商兼容"问题，华为、OPPO、小米均有类似的额外权限管控机制，建议在报告的兼容性章节中说明。

## 五、构建与分发

### GitHub Actions 自动构建

项目配置了 `.github/workflows/build.yml`，每次 `git push` 后自动：
1. 在 Ubuntu 环境安装 JDK 17、Android SDK/NDK、Buildozer
2. 执行 `buildozer -v android debug` 编译 APK
3. 将 APK 上传为 Artifacts，可在 Actions 页面直接下载

### 扫码安装

1. 从 GitHub Actions 下载最新 APK
2. 发送到手机（微信/QQ/网盘均可）
3. 在手机文件管理器中点击安装（需开启"允许安装未知来源应用"）
